# ToDonut Feature Audit

Authority: `DESIGN.md` current launch requirements.

The original audit was read-only. The 2026-07-02 closure sections record subsequent production, test and documentation corrections. The desktop cleanup pass added the forward-only Supabase migration `0005_fix_snapshot_operation_cleanup.sql`.

## 1. Executive summary

ToDonut is not launch-ready. The implementation has a broad working application foundation: canonical snapshot persistence, optimistic mutation coordination, feedback, modal primitives, task views, Task editor, hierarchy domain commands, Reference Lists, Projects/Areas, configuration records, recurrence schedules, Trash, Diagnostics and desktop export are materially present. The remaining launch risk is concentrated in final authentication/recovery, owner-only Supabase access enforcement, and several hierarchy/user-workflow gaps where domain commands exist but complete user-facing workflows are absent or incomplete.

Finding counts:

| Priority | Count |
| --- | ---: |
| P0 launch blocker | 2 |
| P1 major launch gap | 4 unresolved; 3 resolved in closure passes |
| P2 minor launch gap | 0 confirmed code gaps; responsive verification remains pending |
| P3 polish or low-risk gap | 3 |
| Verification pending | 14 |

Major systems materially complete from code inspection: shared canonical mutation path, first-save-wins revision checks, failure recovery panel, desktop export guard, Settings Diagnostics, Task editor basics/dates/checklist persistence, recursive hierarchy rendering, Reference List create/edit/archive/delete/restore, Project/Area create/edit/detail/progress/lifecycle, configuration records, Trash modes, and application-level recurrence catch-up.

Major incomplete systems: final authentication and recovery, owner-only Supabase access enforcement, full hierarchy movement/reorder UX, safe operation-specific Undo coverage, Must Do Today editing, list detail route updates from the Lists browser, list bulk multiline entry UI, Task tag filter UI, movement impact confirmations, and server-authoritative recurrence processing/concurrent-tab handling.

Known launch blockers:

- `AUTH-01`: authentication is password-only and lacks final TOTP/recovery/lockout/password-reset flows.
- `RLS-01`: Supabase policies/RPCs allow any authenticated user to access or replace the singleton owner snapshot.

Manual signed-in responsive testing is still required. This audit records pending visual/runtime verification separately and does not treat lack of manual testing as a defect.

Checks run:

- `npm.cmd run typecheck`: passed.
- `npm.cmd run build`: passed. Vite reported the existing chunk-size warning for the main JS bundle.
- Focused tests: 35 passed across Task views/hierarchy/bulk semantics, recurrence, Status defaults, lifecycle, export and Undo.

## 2026-07-02 focused launch-gap implementation update

- `TASK-01`: complete in code and focused tests. Must Do Today is in Task drafts, Create/Edit UI, save Activity values and Today selector coverage; aggregate parents keep it inactive.
- `LIST-01`: complete in code. The ordinary Lists browser now pushes `#/lists/:listId`; Back uses browser history with a `#/lists` fallback, and existing route restoration handles refresh/Forward. Signed-in browser verification remains pending.
- `LIST-02`: partially complete. Single/Multiple Item UI, trimming, blank-line removal, order preservation, atomic mutation and read-only rejection are implemented and tested. Operation-specific multiline Undo remains outstanding.
- `HIER-01`: complete in code for shared single/bulk Move, root and Parent destinations, subtree inheritance and central domain validation. Signed-in responsive verification remains pending.
- `HIER-02`: complete in code for drag-only sibling ordering with a Lucide handle, pointer/touch drag events and focused Up/Down arrow support. Dragging calls sibling-only commands and cannot reparent.
- `UNDO-01`: materially improved but not complete. Move, reorder, Task Trash, Project deletion, Area deletion and affected-record bulk Undo are wired without whole-application snapshot restoration. Full operation-specific aggregate cascade and individual ancestor-reopen Undo verification remains outstanding.
- `REC-01`: resolved by binding launch decision. Authenticated startup/focus/visibility/foreground catch-up, deterministic occurrence keys and canonical revision protection are accepted as launch-complete; no server processor is required.

P2, P3 and verification-pending findings remain in force unless explicitly superseded above.
